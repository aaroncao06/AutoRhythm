How It Works

9-phase pipeline. AI handles guide generation and alignment. Everything after that is fully deterministic — no neural model touches the human voice.

Phase 0: NormalizeDeterministic
Resample to 48kHz, mono for analysis
Phase 1: Guide GenerationAI
ACE-Step 1.5 generates a rap vocal from lyrics, Demucs isolates vocals
Phase 2: SyllabifyAI
CMUdict + G2P maps lyrics to canonical syllables
Phase 3: Forced AlignmentAI
Montreal Forced Aligner extracts phone-level timestamps from both vocals
Phase 4: Anchor MappingDeterministic
Map each human syllable anchor to the guide's timing
Phase 5: Clip GroupingDeterministic
Safe-boundary scoring finds acoustically clean split points
Phase 6: Edit PlanDeterministic
Piecewise time-warp specification for every segment
Phase 7: RenderDeterministic
Rubber Band pitch-preserving time-stretch produces the corrected vocal
Phase 8: Audacity SessionDeterministic
Builds a full session with clips, labels, and tracks via mod-script-pipe

Key Features

Zero-Sample Precision

Every rendered syllable anchor lands exactly on the guide anchor at the integer sample index. Not "close enough" — mathematically exact.

Voice Preservation

AI generates a timing reference and extracts timestamps. The human voice is only cut and time-stretched — never regenerated or transformed.

Transparent Editing

Output is a full Audacity session with visible clips, label tracks, and waveforms. Every edit is inspectable and overridable.

Two Modes

Guide mode uses an AI vocal as timing reference. Beat-only mode snaps syllables to the detected beat grid.
